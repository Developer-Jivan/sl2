# 📦 3. Dataset Understanding

* `load_digits()` gives:

  * 8×8 grayscale images
  * pixel values: **0–16**

👉 That’s why we do:

```python
x = x / 16.0
```

✔ Normalization → makes training stable & faster

---

# 🔧 4. Preprocessing

```python
x = x.reshape(-1, 8, 8, 1)
```

Why?

CNN expects:

```
(height, width, channels)
```

* 8 → height
* 8 → width
* 1 → grayscale channel

---

# ✂️ 5. Train-Test Split

```python
train_test_split(...)
```

👉 Purpose:

* Train → learn
* Test → check performance

Without this → model just memorizes

# 🧠 6. CNN Working (VERY IMPORTANT)

```
Image (8x8)
   ↓
Conv2D → detects patterns (edges, shapes)
   ↓
ReLU → removes negative values
   ↓
Pooling → reduces size
   ↓
Flatten → convert to 1D
   ↓
Dense → learning
   ↓
Softmax → probability of each class
```

---

# 🚀 8. Model 2 (Improved CNN)

```python
Conv2D
```

## 🔍 Each Improvement Explained

### 🔹 (1) MaxPooling

```python
layers.MaxPooling2D((2,2))
```

✔ Reduces image size
✔ Keeps important features
✔ Faster computation

---

### 🔹 (2) More Filters (32 → 64)

👉 More filters = detect more patterns

---

### 🔹 (3) Bigger Dense Layer

```python
Dense(128)
```

✔ Learns complex relationships

---

### 🔹 (4) Dropout

```python
Dropout(0.5)
```

👉 During training:

* randomly removes 50% neurons

✔ Prevents **overfitting**

# 📉 9. Training (fit)

```python
model.fit(...)
```

What happens internally:

```
Forward pass → prediction
       ↓
Loss calculation
       ↓
Backpropagation
       ↓
Weights updated
       ↓
Repeat for epochs
```

---

# 📊 10. Validation (IMPORTANT)

```python
validation_data=(x_test, y_test)
```

👉 Tracks:

* training accuracy
* validation accuracy

### Why important?

| Case           | Meaning      |
| -------------- | ------------ |
| Train ↑, Val ↓ | Overfitting  |
| Both low       | Underfitting |
| Both high      | Good model   |
